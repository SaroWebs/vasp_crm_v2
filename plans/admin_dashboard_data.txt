### Admin Dashboard Data Flow & Presentation Analysis

#### Data Flow Architecture
The dashboard employs a hybrid rendering approach:
1. **Initial Page Load (Inertia.js)**: 
   - The user requests the `/admin/dashboard` route.
   - `AdminDashboardController@index` determines the user's role and assigns a `dashboard_type` (`admin`, `manager`, or `employee`).
   - It retrieves the user's permissions and passes these limited initial props (`dashboard_type`, `userPermissions`) to the `dashboard.tsx` Inertia page component.
   - Global authenticated user data (`auth.user` with `roles` and `employee` relations) is provided by the `HandleInertiaRequests` middleware.
2. **Asynchronous Widget Rendering (AJAX)**:
   - Instead of the controller eagerly loading all dashboard statistics (which could be heavy and block the initial render), the dashboard utilizes a modular widget architecture.
   - Individual React components (widgets) manage their own loading state and asynchronously fetch their specific data via AJAX requests (using Axios) after the initial Inertia page load.
   - These AJAX requests hit endpoints like `/admin/api/dashboard/stats`, `/admin/api/dashboard/tickets`, `/admin/api/dashboard/tasks`, etc., which correlate to methods on `AdminDashboardController` (e.g., `getStats`, `getTickets`, `getTasks`). These methods delegate business logic to the `DashboardService`.

#### Data Shown & Presentation
The dashboard dynamically renders different UI components and layouts based on the evaluated `dashboard_type`.

**1. Admin Perspective**
- **Recent Tickets**: Rendered via `RecentTicketsWidget` as a vertical list. It shows the latest submitted tickets with their subject, client name, ticket number, and a colored status badge.
- **Recent Reports**: Loaded via `RecentReportSection`.
- **Attendance**: Displayed using the `AttendanceList` component (with filtering and pagination enabled) alongside the top widgets.
- **Major Tasks**: Displayed via the `MajorTasks` component, showing significant organizational tasks.
- **Task Timeline**: A chronological feed displaying activity and status changes across tasks, provided by `TaskTimeline`.
- **Employee Task Progress**: A dynamic card (`EmployeeTaskProgress`) allowing administrators to select a specific employee (or "All") from a dropdown to view time entries and task progress.
- **Status Distributions**: Uses cards to display "Ticket Status Distribution" and "Task Status Distribution" with badge counts. (Note: In the current implementation, these rely on `ticketStats` and `taskStats` Inertia props which may need to be wired up or fetched asynchronously).

**2. Manager Perspective**
- **Stats Summary**: Displayed at the top via `DashboardStatsWidget`, showing key metrics in a card grid (Team Members, Department Tasks, Tasks Due This Week, Overdue Tasks).
- **Department Tasks**: A list card displaying active tasks within the manager's departments, including title, department name, due date, status, and priority badges.
- **Department Summary**: Displays metrics per department, showing pending tasks and tasks completed this month.
- **Team Workload**: Shows individual team member capacity, displaying their task count and current status (e.g., overloaded, busy) with visual badges.
- **Team Activity Timeline**: `TaskTimeline` component scoped to the manager's team flow.
- **Quick Actions**: Prominent buttons granting fast access to Tasks, Tickets, and Departments pages, conditionally rendered based on `userPermissions`.

**3. Employee Perspective**
- **Personal Stats**: Top-level cards (`DashboardStatsWidget`) showing "Due Today", "Overdue", "In Progress", and "Completed This Month".
- **Task Summary**: Handled by `DashboardTasksWidget`.
- **Attendance Calendar**: A visual calendar (`AttendanceCalendar`) summarizing the employee's attendance over the current period.
- **Task Board**: An interactive, Kanban-style board (`Board` component) that fetches tasks assigned specifically to the employee and allows for drag-and-drop workflow progression between statuses.
- **Employee Activities & Reports**: Feeds for personal activities and recent reports.
- **Workload Matrix**: A card with a quick summary and a link to the detailed workload matrix page.
- **Personal Task Timeline**: Status changes and updates specifically for the employee's assigned tasks.

#### Summary
The dashboard's design is heavily componentized, favoring lazy-loading data through specialized widgets to keep the initial page load fast and snappy. The UI uses modern layout patterns (CSS grids/flexbox), clear typography, and conditional styling (e.g., destructive badges for overdue or open items, warning colors for upcoming deadlines) to present dense, role-specific information efficiently without overwhelming the user.
