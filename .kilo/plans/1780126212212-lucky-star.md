# My Attendance Page Implementation Plan

## Overview
The "My Attendance" page (`/my/attendance`) is a React/Inertia page for employees to view their attendance records, punch entries, leaves, and remote work status. Currently, the page has placeholder content in tabs and relies on `DailyAttendancePanel` and `AttendanceCalendar` components.

---

## Control Logic & Process Analysis

### Backend Flow
1. **Authentication**: User must be authenticated via Fortify
2. **Employee Lookup**: `Auth::user()->employee` links user to employee record
3. **Employee Code Required**: Attendance operations require `employee->code` (biometric ID)

### Data Sources
| Source | Model | Purpose |
|--------|-------|---------|
| `Attendance` | `app\Models\Attendance` | Daily punch records (punch_in, punch_out) |
| `Punch` | `app\Models\Punch` | Raw biometric punch data |
| `LeaveRequest` | `app\Models\LeaveRequest` | Leave requests with status (pending/approved/rejected) |
| `RemoteWorkRequest` | `app\Models\RemoteWorkRequest` | Remote work requests |
| `FieldWorkAssignment` | `app\Models\FieldWorkAssignment` | Field work assignments |
| `Shift` + `EmployeeShiftAssignment` | `app\Models\Shift` | Shift schedules |
| `Holiday` | `app\Models\Holiday` | Holiday calendar |

### Backend APIs (routes/api.php)
| Endpoint | Method | Purpose |
|----------|--------|---------|
| `/api/my/attendance` | GET | Monthly attendance records + summary |
| `/api/my/attendance/punch` | POST | Record punch-in/out |
| `/api/daily/attendance` | GET | Daily attendance details (admin) |

### Attendance Calculation Service
`App\Services\AttendanceCalculationService` handles:
- Leave detection (`isEmployeeOnLeave`)
- Remote work detection (`isEmployeeOnRemoteWork`)
- Field work detection (`isEmployeeOnFieldWork`)
- Shift resolution with grace periods
- Holiday/non-working day detection

## Implementation Plan

### Phase 1: Details Tab - Today's Status Dashboard

**Goal**: Show today's attendance status, punch status, and quick actions.

**Data Required**:
- Today's attendance record (punch_in, punch_out, mode)
- Current shift details (start_time, end_time, grace_minutes)
- Whether employee is on leave today
- Whether employee is on remote work today
- Whether employee is on field work today

**Components to Create/Use**:
- `TodayAttendanceStatus` - Display current day's status
- `PunchWidget` - Already exists, needs integration
- `AttendanceSummaryCards` - Already exists, can show today's summary

**API Calls**:
```tsx
// Fetch today's single record
GET /api/my/attendance?month=<current>&year=<current>
// Filter for today's record from response
```

### Phase 2: Calendar Tab Enhancement

**Goal**: Interactive calendar with status indicators.

**Current State**: `AttendanceCalendar` component exists and works.

**Enhancements Needed**:
- Add today marker
- Add click-to-navigate functionality
- Show status indicators (present, late, leave, remote, holiday)

### Phase 3: Leaves Tab Implementation

**Goal**: Display leave history and request form.

**Data Required**:
- `LeaveRequest` records for the employee
- `LeaveType` for leave type names
- `LeaveBalance` for available leaves

**Components to Create**:
- `LeaveHistoryList` - Table of past leave requests
- `LeaveRequestForm` - Form to request new leave
- `LeaveBalanceDisplay` - Show available leave balances

**API Endpoints Needed**:
```php
// New endpoints to create
GET  /api/my/leaves              // List employee's leaves
POST /api/my/leaves              // Request new leave
GET  /api/my/leave-balances      // Get leave balances
```

### Phase 4: Remote Work Tab Implementation

**Goal**: Display remote work history and request form.

**Data Required**:
- `RemoteWorkRequest` records with status
- Approval workflow status

**Components to Create**:
- `RemoteWorkHistoryList` - Table of past/future remote work
- `RemoteWorkRequestForm` - Form to request remote work

**API Endpoints Needed**:
```php
// New endpoints to create
GET  /api/my/remote-work              // List employee's remote work
POST /api/my/remote-work              // Request new remote work
```

## Key Considerations

- **Employee Code Check**: Must verify employee has a biometric code before allowing punch
- **Status Determination**: Use `AttendanceCalculationService.determineAttendanceStatus()` logic
- **Leave Balances**: Must be year-based, fetched from `LeaveBalance` model
- **Remote Work**: Only show "approved" status for active remote work
- **API Error Handling**: Show appropriate messages when employee code is missing
