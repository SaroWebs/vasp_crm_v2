# Leave Management: Convert from Hourly to Leave Count System

## Overview

Convert the leave management system from tracking `allocated_hours`/`used_hours` to tracking `assigned_leaves`/`consumed_leaves` as integer day counts.

## Current State

- **Database**: `leave_balances` table uses `allocated_hours`, `used_hours`, `opening_balance`, `closing_balance` (decimal, 2 decimals)
- **Model**: `LeaveBalance` model has computed attributes `remaining_hours`, `carried_over_hours`, `remaining_leaves`, `carried_over_leaves`
- **Controllers**: Both `LeaveBalanceController` and `LeaveController` handle hour-based calculations
- **Frontend**: UI displays values as "leaves" but stores as hours internally

## Proposed Changes

### 1. Database Migration

Create a new migration to rename and modify columns:

| Current Column | New Column | Type | Description |
|----------------|------------|------|-------------|
| `opening_balance` | `opening_leaves` | integer | Carry-over leaves from previous year |
| `allocated_hours` | `assigned_leaves` | integer | Leaves assigned for the year |
| `used_hours` | `consumed_leaves` | integer | Leaves consumed |
| `closing_balance` | `remaining_leaves` | integer | Available leaves |

## Considerations

### Data Migration

- Assume 8-hour workday for conversion: `days = hours / 8`
- Round down to whole days (partial days are not tracked)
- `opening_leaves` and `remaining_leaves` should be integers

### Business Logic

- Leave requests already use `getDaysCount()` which returns whole days
- No change needed to `LeaveRequest` model
- Closing balance formula: `opening_leaves + assigned_leaves - consumed_leaves = remaining_leaves`

### Backward Compatibility

- API responses will change field names
- Any external integrations will need to update
